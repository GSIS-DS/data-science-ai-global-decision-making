# AI Verification and Text-Data Exercise

## Purpose

Integrate text-data analysis with responsible AI verification workflows.

## Core Tasks

1. Perform a basic text-data task using provided or approved data.
2. Evaluate at least one AI-assisted output (code, summary, labels, or interpretation).
3. Verify claims and outputs against underlying data and documented logic.
4. Record material AI use using the required AI Use Statement fields.
5. Report what was accepted, revised, or rejected and why.

## Deliverables

- Reproducible notebook/report
- Verification log
- AI Use Statement

## Scope and Timing

- Released: Week 12
- Due: Week 14, on the date published in the LMS
- Use the prepared text collection or another source explicitly approved for the exercise.
- No paid API is required. A prepared model response will be supplied so access to an external AI service is not necessary.

## Evaluation Criteria

| Criterion | Weight |
| --- | ---: |
| Transparent text-data method and documentation | 25% |
| Quality of checks against source text or data | 30% |
| Identification and correction of errors or unsupported claims | 20% |
| Interpretation and limitations | 15% |
| Complete AI Use Statement and reproducibility | 10% |

## Weight

15% of the course grade as one integrated component.
