# Project Repository Template

Recommended structure for capstone project repositories.

```text
project-root/
├── README.md
├── AI_USE.md
├── data/
│   ├── raw/
│   └── processed/
├── notebooks/
├── outputs/
├── src/
└── docs/
```

## README Should Include

- Project title and team
- Decision question
- Data sources and limitations
- Summary of method
- Key findings and caveats
- Run instructions
- Dependency and environment information
- Deliverable map linking the memo, visuals, data dictionary, and AI Workflow Card
- Team roles and contribution record

Do not commit API keys, personal data, confidential files, or third-party data that cannot legally be redistributed. When data cannot be included, provide documented retrieval steps and a stable source identifier.
