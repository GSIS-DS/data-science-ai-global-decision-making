# Course Schedule

| Week | Topic | Learning Objective | Learning Content / Method | Assignment / Activity |
| --- | --- | --- | --- | --- |
| 1 | Data, AI, and Evidence for Global Decision-Making | Distinguish evidence, interpretation, and decision claims in global contexts. | Course framing, notebook workflow, introductory data preview. | In-class interpretation activity; set up GitHub and Colab. |
| 2 | Working with Data in Python | Load, inspect, and summarize tabular global indicators. | pandas basics, filtering, descriptive summaries. | DRT 1; small CSV inspection notebook. |
| 3 | Cleaning and Transforming Data | Identify common quality issues and document corrections. | Missingness checks, duplicates, type fixes, derived variables. | DRT 2; cleaning log exercise. |
| 4 | Combining and Reshaping Global Data | Combine related tables and reshape data for analysis. | Merge keys, joins, unmatched rows, wide/long transformations. | Guided merge lab; project-question brainstorming. |
| 5 | Exploratory Analysis and Visualization | Build and critique exploratory visuals for cross-country comparison. | Visual encodings, uncertainty, communication design basics. | DRT 3; visualization critique and revision; Mini-project 1 preview and examples. |
| 6 | Global Data Sources and Reproducibility | Evaluate source quality and strengthen reproducibility practices. | Source documentation, provenance, folder and workflow discipline. | Mini-project 1 brief released; Data Readiness Task 4. |
| 7 | Statistical Reasoning for Applied Analysts | Apply association and uncertainty concepts carefully. | Comparative reasoning, variation, interpretation limits. | Applied analysis workshop. |
| 8 | Midterm: Individual Applied Data Analysis | Demonstrate individual competency on a new manageable dataset. | Timed applied analysis with documented reasoning and reproducibility. | Graded midterm individual applied data-analysis assessment; Mini-project 1 due. |
| 9 | Prediction for Decision-Making | Build and evaluate introductory prediction workflows. | Train/test logic, baseline models, simple diagnostics. | Model-evaluation notebook. |
| 10 | Interpretable Machine Learning and Ethical Limits | Explain model behavior and identify practical ethical limits. | Feature interpretation, caveats, responsible deployment framing. | Mini-project 2 brief released; capstone teams and preliminary topic selection confirmed. |
| 11 | Text as Data | Use structured workflows for basic text-data tasks. | Tokenization concepts, categorization logic, transparent documentation. | Text-data lab. |
| 12 | LLMs as Analytical Assistants | Use LLM support while preserving verification discipline. | Prompt design, claim checking, reproducible AI-use documentation. | AI Verification and Text-Data Exercise released; verification workshop. |
| 13 | Bounded Agents and Workflow Design | Design bounded workflows with explicit tools and oversight. | Structured workflow templates and checkpoint-based validation. | Mini-project 2 due; bounded-agent workflow prototype; capstone progress check; AI Workflow Card draft. |
| 14 | From Analysis to Decision Communication | Translate analysis into decision-ready outputs. | Executive messaging, uncertainty communication, memo drafting. | AI Verification and Text-Data Exercise due; capstone draft peer review. |
| 15 | Capstone Studio | Finalize integrated technical and communication deliverables. | Revision studio, reproducibility check, rehearsal feedback. | Final capstone repository, reproducible notebook or report, data dictionary, executive memo, and AI Workflow Card due; presentation rehearsal and final peer feedback. |
| 16 | Final: Global Data Challenge Presentations | Present and defend project decisions and methods. | Team presentations, peer review, individual defense/reflection. | Capstone presentation; peer review; individual capstone defense/reflection. |
